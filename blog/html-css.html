Relative, Absolute and Fixed positioning in CSS

What are they and how do they differ?

08 September 2021

Every element on a web page is a block or a rectangle of pixels. Relative, absolute and fixed positioning are some of the tools you can use in CSS to move these blocks exactly where you want them to go (instead of having one big list of elements like the below).

This initial positioning is called "static" and is the default for every element.

Relative Positioning

Relative positioning is a way to move your blocks (or elements) in context to themselves. The movement commands you use in conjunction with relative positioning will operate based on where the block would normally be.

In the above example the box would move down 600 px or pixels from its original position. When using relative positioning the moved block is not removed from the documents flow - ie. in the below picture box 2 and 3 will still act as though box 1 is in its original position even if it has moved.

Absolute Positioning

Absolute positioning is a way to move blocks in context to the closest containing element that has been assigned a position (eg. relative). The containing element is referred to as the 'parent' element.

In this example the box would move 600 px or pixels down from the the top edge of the parent. This means that if the box was already positioned 700 pixels below the parents top edge it would actually move up up 100 pixels from its current position.

You can use this function to make the box move outside of the parent by specifying a negative (eg. -10px). You can also make the box flush with the parent container by using 0px as the distance - the below would have the box flush with the parents left wall.

When using the absolute positioning function elements are removed from the documents flow - when box 1 moves box 2 and 3 will shift up the queue.

Fixed Positioning

Fixed positioning is basically the same as absolute positioning except that its 'parent' element is the viewport (browser window). This means that if you resize your browser window or scroll the block will move with it - which means it can sit in front of the other content if you don't plan for enough space. Fixed positioning can be helpful if you want something like a naviagtion bar to be visible at all times. Just like absolute positioning the fixed position block is removed from the documents flow.
