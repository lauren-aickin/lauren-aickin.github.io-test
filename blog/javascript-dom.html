Javascript

An introduction from a programming noob, for programming noobs

Javascript is an important part of a coders toolkit, adding life and movement to our websites. It may seem like a foreign language right now but it is one well worth learning!

So how does this magical Javascript fit in with HTML and CSS within the web developer toolkit? Lets use the context of film as an analogy: HTML would be the script, CSS the costume and Javascript the actor.

Basic Ideas/Lingo

In Javascript strings can be used to store characters - such as letters, numbers and words. Strings have to be within single or double quotations (either is fine as long as you use the same at both ends).
Values are another basic component - they can be many things such as:
an integer (int) 10,
a string "pancakes"
a function() {}
or more that you may learn along the way!
Variables or var is what we use to store these values. eg, var name = val or var lauren = "Hates pineapple pizza".

Functions

Functions are one of the core Javascript values to get your head around. Functions are just what they sound like on the tin - they're a procedure/block of code that performs a certain task. Functions are executed when something invokes it (like a magician invoking a spell).
This means you can code a task that needs to be used multiple times just once in a function block and then use it to your hearts content by calling its name. eg.
function addition(num1,num2) { let res = num1 + num 2 return res; }
would be called by typing:
addition(5,1)
and would return the answer 6.

Control Flow and Loops

One of Javascripts strongest features is control flow. This is the order a program's code executes and is done through a few different statement stypes - today we'll focus on looping.
Looping is used when you want to tell the computer to repeat a task over and over again - this can be a for loop to loop through the code block a specified number of times or a while loop that loops until a certain condition is met.
Lets look at some real life examples as analogys.
If you were following an exercise plan that specified 20 squats you would use a for loop to repeat the movement 20 times.
If you were following an exercise plan that specified "you must be able to lift 20KGs before moving on" you would repeat the exercise with a while loop until the 20kgs has been achieved.

Arrays and Objects

Arrays and objects are two similar concepts that are used within javascipt for different purposes. They are both used to store a collection of values and are changeable(mutable) so what are their differences?

Objects are used to represent a "thing". That could be an album, a building/structure, a colleague or a flower - anything that can be defined by a set of characteristics (properties). These properties contain a key (eg. age) and a value (eg. 26).
To access the data within an object we use a dot or a bracket. For example in our 'lemon' object to grab the value of the taste key we could either use:
lemon.taste or lemon['taste']
These both work but if there are spaces in your key you'll need to use the bracket variant.

Arrays are used to create and store lists of items in one variable. They are ordered by their numerical position in the list and can be accessed by calling their index number. These lists always start at zero - so if you want to access the first item in the following array:
var sillyFoods = ['sausage', 'salmon', 'stroganoff'];
you would write:
sillyFoods[0];
which would return 'sausage'.

The DOM

The DOM (Document Object Model - but you don't have to remember the full name!) connects web pages to scripts or programming languages (eg. javascript!) by representing a documents structure. It does this with a tree structure to show the relationships between different elements (called nodes) much like you would see in a geneology family tree.
You won't see this structured exactly as a tree in practise but it's good to know that that's how elements are linked and categorised. What you will use in practice are the browsers rendition of the DOM through the developer tools - the most commonly used is chromes.
You can access the Dev tools either by clicking the 3 dots in the top right of your browser > more tools > Developer Tools, with ctrl shift i or by right clicking and hitting inspect.
What you do in here will not directly affect the source code - which means we can play to our hearts content with no fear of breaking a website!
This is helpful for testing out bug fixes/code lines before commiting to changing your source code - or just messing with your friends! Keep in mind that when you refresh/leave the page all changes will be reverted to the original code.
